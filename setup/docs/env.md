# Environment Variable Management

## Boot-to-Desktop Flow

```
tty1 login
  └─ .profile
       ├─ sources .bashrc
       ├─ adds ~/bin, ~/.local/bin to PATH
       ├─ sources ~/.local/bin/env (uv), ~/.cargo/env
       ├─ sources .vars → .secret-vars → .local-vars
       └─ if no DISPLAY on tty1: exec startx
            └─ .xinitrc
                 ├─ xrdb, setxkbmap, xset
                 ├─ sources .vars → .secret-vars → .local-vars (again, for X process)
                 ├─ exports XDG_SESSION_TYPE=x11, XDG_CURRENT_DESKTOP=LeftWM
                 ├─ systemctl --user import-environment (pushes shell env → systemd)
                 ├─ dbus-update-activation-environment --systemd DISPLAY XAUTHORITY
                 ├─ starts graphical-session{-pre,}.target
                 └─ dbus-run-session leftwm
                      └─ themes/current/up
                           ├─ sources .profile (for leftwm child env)
                           ├─ starts dunst, polybar
                           └─ user runs lwup to launch workspace apps
```

## Three-Layer Variable System

| Layer | File | Source | Content |
|-------|------|--------|---------|
| 1 | `~/.vars` | diencephalon | Base paths, tool config, non-secret app vars |
| 2 | `~/.secret-vars` | hypothalamus/secrets | API keys, tokens |
| 3 | `~/.local-vars` | hypothalamus/local-dotfiles/host-$(hostname) | Machine-specific overrides (GPU, hardware, ports) |

Each layer can reference variables from previous layers. Loaded in order by `.profile` and `.xinitrc`.

## systemd Integration

### environment.d (static, for services)

`restow` auto-generates `~/.config/environment.d/{10,20,30}-*.conf` by shell-expanding each layer. These are loaded by `systemd --user` at manager startup and provide the base environment for all user services.

**Limitation:** environment.d files are read once at systemd user manager start (login). Changes require `systemctl --user daemon-reload` and restarting affected services, or a full re-login.

### import-environment (dynamic, at X start)

`.xinitrc` runs `systemctl --user import-environment` after X starts. This pushes the current shell environment (which now includes DISPLAY, XAUTHORITY, and the XDG overrides) into the systemd user manager, making these available to all subsequently started services.

### dbus-update-activation-environment

`.xinitrc` also runs `dbus-update-activation-environment --systemd DISPLAY XAUTHORITY` to ensure dbus-activated services get display access.

## tmux Environment

tmux captures the environment when its **server** starts (first session creation). The `update-environment` option (default includes DISPLAY, XAUTHORITY) propagates from the **attaching client** to the session on `attach-session` or `new-session`.

**Problem scenario:** lwup creates `main-bg` session before or without a client that has DISPLAY set. The tmux server inherits an env without DISPLAY, and `new-window` does not trigger `update-environment` — only attach does.

**Fix:** `.tmux.conf` adds session-level vars via `update-environment`. lwup sets the tmux global env explicitly after session creation:

```
tmux set-environment -g DISPLAY "$DISPLAY"
tmux set-environment -g XAUTHORITY "$XAUTHORITY"
```

## XDG_SESSION_TYPE

When logging in via tty1 and running `startx`, logind sets `XDG_SESSION_TYPE=tty` and never updates it. Some applications (including snap-confined ones) may check this value.

**Fix:** `.xinitrc` exports `XDG_SESSION_TYPE=x11` before `import-environment`. Note: this only affects the environment, not the actual logind session type (which remains `tty` in `loginctl`).

## Headless Stations

On headless machines (no X, no leftwm):

- `.profile` sources vars but does not `exec startx` (no tty1 or DISPLAY already set)
- Services rely solely on `environment.d` files generated by `restow`
- No DISPLAY, XAUTHORITY, or XDG_SESSION_TYPE in environment
- tmux sessions have no display-related env vars
- Snap GUI apps are not relevant

Ensure `restow` is run after any changes to `.vars`, `.secret-vars`, or `.local-vars`, followed by `systemctl --user daemon-reload`.

## Debugging Checklist

```bash
# What does systemd user manager see?
systemctl --user show-environment | grep -E 'DISPLAY|XDG_|DBUS_|XAUTH'

# What does the current shell see?
echo "DISPLAY=$DISPLAY XAUTHORITY=$XAUTHORITY XDG_SESSION_TYPE=$XDG_SESSION_TYPE"

# What does tmux server see?
tmux show-environment | grep -E 'DISPLAY|XAUTH'

# What's the login session type?
loginctl show-session $(loginctl --no-legend | awk '{print $1}') -p Type

# Are environment.d files current?
ls -la ~/.config/environment.d/
cat ~/.config/environment.d/10-vars.conf | head -5

```
